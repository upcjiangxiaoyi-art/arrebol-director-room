# Arrebol D 暗河红霞导演系统

SillyTavern RP 场外导演插件。副 API 读取最近 N 轮剧情，输出情感 / 剧情方向，注入正文供主模型（如 Gemini）执行。

> 设计理念：长上下文给导演，短方向给演员。给“在场感极强但容易发疯”的主模型，配一个理智的场外导演。

---

## 核心特性

- **双导演**：情感导演 + 剧情导演，各自独立配置、独立预设、独立注入，互不抢同一层。
- **手动台**：选复盘范围 → 填补充要点 → 生成方向 → 预览可改 → 注入正文。
- **自动触发**：每累积约 N 个助手正文轮次，自动调副 API 生成导演方向并注入，无需手动点。
- **严格结算**：开启自动注入时，只有「API 成功 **且** 注入成功」才算这一拍完成、才推进进度。中途失败不会白丢。
- **失败会说话**：API 欠费 / 中转站炸 / 网络断 / 注入失败时，第一次会弹窗告知你，并明确「这一拍不会丢，稍后自动重试」。
- **静默退避重试**：失败保拍后，插件会在后台按 45s → 90s → 120s → 180s 的退避节奏自动重试，修好 API 后会自己补上，不刷屏。
- **每聊天独立计数**：进度按聊天隔离，切换角色卡、刷新页面、重 roll 楼层都不会乱。

## 安装

扩展面板 → Install extension → 填仓库地址。装完刷新页面。

## 使用

### 手动模式

1. 点聊天框旁的 🎬 快捷入口打开面板
1. 「配置」区填 API（OpenRouter 填 `https://openrouter.ai/api/v1`）、加载模型、选模型
1. 选复盘轮数（最近 10 / 20 / 30 / 50 / 自定义）
1. 「手动补充」填角色卡要点、世界书、当前担心
1. 「情感 / 剧情导演预设」可改可不改（有内置默认）
1. 点「生成方向」，结果出现在「导演方向」区
1. 不满意可手动改，或写补充指令点「重新分析」
1. 满意后点「注入正文末尾」，下一轮主模型就能读到

### 自动模式

1. 在对应导演下打开「启用自动触发」
1. 设「触发间隔」N（每累积约 N 个助手正文轮次触发一次）
1. 确认「自动注入」已开（否则只生成不注入）
1. 之后正常对话即可：到点自动生成、自动注入
1. 「自动分析前显示提示」可关——关掉的只是开始提示，**故障告警不受影响，始终会报**

---

## 计数与结算说明

- 进度按「当前助手正文楼数 − 上次结算点」计算，全量回头数真值，不是只进不退的事件计数。
- 因此：重 roll、补档、删楼、切卡、刷新，进度都会跟着现实自动对齐，不会漂移。
- 一拍只在「成功生成 + 成功注入」后才结算并推进，失败会保住这一拍、稍后自动重试。

---

## 已知行为

- **触发层重新生成不会二次触发**：某一层「正好触发」并成功注入后，如果你对这一层 AI 回复不满意、直接重 roll 它，导演**不会**自动再触发一次（这一拍已结算）。
  这是当前设计的有意取舍：要让它自动重触发，需引入与「半截读数」无法区分的回退判断，会动摇整套计数稳定性，得不偿失。
  **如需对该层重新出方向，请手动点「重新分析」+「注入」即可，10 秒搞定。**
- 普通楼层（未触发层）的重 roll / 补档完全正常，进度会自动回弹。
